Work Experience

Amazon AWS SageMaker (Software Development Engineer Intern)

In my first project I onboarded OpenID Connect (OIDC) configurations to the CloudFormation (CFn) Workteam resource. CFn abstracts away the complexity of specific resource APIs, and OIDC configurations allows customers to manage and authenticate group workers directly through the external identity provider of their choice without managing password files. The CFn Workteam resource originally only supported Amazon Cognito Identity Providers and the key deliverable of this project was to add support for identity providers that adhere to the OpenID Connect standard to give customers more flexibility for managing their workers. In my second project I added support for custom scope and static parameters to OIDC configurations within the Workforce resource. Custom scope in OIDC adds an extra layer to the identity provider to differentiate the authentication results between various clients effectively increasing flexibility in the authentication process. The additional static query parameters contribute to the authentication/token endpoints by returning a set of user attributes based on the query parameter defined. Additionally, found a bug in the original implementation of the static parameters that inadvertently capitalized/lowercased parameter keys when moving the object between APIs. Squashed bug by internally changing datatype from a map to a list of objects.
